## 2. 온보딩 (`/onboarding`)

### 공통 레이아웃
- 상단: 프로그레스 바 (4스텝 기준, `bg-brand` fill)
- 뒤로가기 버튼 (← 아이콘, Step 0에서는 숨김)
- 캐릭터 말풍선 (CharacterBubble): 왼쪽 캐릭터 원 + 오른쪽 말풍선
- 하단: CTA 버튼 (조건부 활성화)

### Step 0: 이름
- 캐릭터: 물결이(水) `#89B0CB`
- 말풍선: "반가워요! 이름이 뭐예요?"
- 인풋: 이름 (2~10자)
- CTA: "다음" (입력 시 활성화)

### Step 1: 성별
- 캐릭터: 물결이(水)
- 말풍선: "{name}님, 성별을 알려주세요"
- 선택지: 남성 / 여성 (2개 버튼, 수평 배열)
- **자동진행**: 탭 후 0.3초 뒤 다음 스텝

### Step 2: 생년월일 + 시진
- 2단계 합침 (웹에서는 한 화면)
- 캐릭터: 물결이(水)
- 말풍선: "생년월일과 태어난 시간을 알려주세요"
- 날짜: `<input type="date">` 또는 커스텀 DatePicker
- 시진: 12지지 시진 그리드 (3열 4행) + "모르겠어요" 옵션
  - 자시(23:00~01:00) ~ 해시(21:00~23:00)
- CTA: "다음" (날짜 선택 시 활성화, 시진은 선택사항)

### Step 3: 사진
- 캐릭터: 불꼬리(火) `#D4918E`
- 말풍선: "마지막으로 정면 사진을 찍어주세요!"
- 보조텍스트: "관상 분석에 사용돼요. 정면 사진이 정확해요!"
- 파일 업로드 버튼 (카메라 아이콘)
- 촬영/선택 후 미리보기 표시
- CTA: "완료" → 결제벽으로 이동

## 7. 공유 페이지 (`/share/[id]`)

### SSR (Server Component)
- `generateMetadata()`로 OG 태그 동적 생성
- 사주/관상 결과 데이터 fetch

### OG 메타데이터
```typescript
export async function generateMetadata({ params }): Promise<Metadata> {
  const profile = await fetchProfile(params.id)
  return {
    title: `${profile.name}님의 사주 결과 | momo`,
    description: `${profile.animalType} · ${profile.sajuSummary}`,
    openGraph: {
      images: [{ url: `/api/og/${params.id}` }], // 동적 OG 이미지
    },
  }
}
```
