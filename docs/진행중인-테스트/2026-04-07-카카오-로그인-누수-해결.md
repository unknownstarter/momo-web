# 카카오 로그인 → profiles INSERT 누수 해결

**시작일:** 2026-04-07
**상태:** 🟡 측정 중
**관련 PR:** [#24 Stage 1](https://github.com/unknownstarter/momo-web/pull/24), [#26 Stage 2](https://github.com/unknownstarter/momo-web/pull/26)
**관련 커밋:**
- Stage 1: `1f560eb` (닉네임 화면 환영 캐릭터 추가)
- Stage 2: `5e93aae` (카카오 로그인을 생시 입력 직후로 이동)
- 광고 URL 옵션 A (예정): TBD

## 가설 (Hypothesis)

> **카카오 로그인을 사용자 정보 입력 직후로 옮기면**, 사용자가 닉네임/성별/생년월일/생시 4개를 이미 입력한 sunk cost 상태에서 로그인을 결정하므로 거부감이 줄어들고, **카카오 로그인 시작자 → profiles INSERT 도달률이 약 61% → 85%+로 개선될 것**이다.
>
> 추가 가설: **광고 destination URL을 `/onboarding`으로 직행시키면**, 랜딩 페이지(`/`)에서 발생하던 이탈(132명, 58.7%)도 구조적으로 사라져 **광고 클릭 → profiles INSERT 도달률이 추가로 1.5~2배 개선**될 것이다.

---

## 배경 (Why now)

2026-04-07 GA4 funnel 분석에서 발견한 누수 (시크릿 창, 모바일 데이터):

```
view_main                            225명 ┐
                                            │ 이탈 -132 (58.7%)  ← 1순위 누수
click_cta_in_main                     93명 ┘
                                            │
start_login                           62명 ┐
                                            │ 이탈 -38 (61%)     ← 2순위 누수 (Stage 2 타겟)
사진 통과 (click_next_in_..._photo)   24명 ┘
                                            │
분석 시작 (click_start_analysis)      20명
```

**문제:**
1. 광고로 들어온 사람의 절반 이상이 랜딩 페이지에서 이탈 (132명)
2. 카카오 로그인 시작한 사람의 61%가 profiles INSERT(사진 업로드) 단계까지 도달 못 함 (38명)
3. **결과: 카카오 OAuth는 통과했지만 profiles에는 한 줄도 없는 "유령 유저"** 가 매일 발생

**원인 (가설):**
- 로그인이 너무 앞단에 배치 → 사용자 가치 인식 전 거부감
- 광고 카피와 랜딩 페이지 사이 mismatch → 즉시 이탈
- 카카오 OAuth 왕복 후 또 14단계 입력해야 함 → 피로감

## 변경 사항 (What changed)

### Stage 1 — 닉네임 화면 환영 카피 (#24, 2026-04-07 오후 배포)

- `/onboarding` Step 0 입력 필드 위에 큰 헤딩 추가:
  > "momo가 사주와 관상을 보기 위해 당신에 대해 궁금해요!"
- 입력 필드 아래에 namuri 캐릭터 GIF + 환영 메시지 추가:
  > "꼭 실명이 아니라도 괜찮아요. / 만세력부터 연애운까지! / 당신과 딱 맞는 사람을 알려줄게요."
- 첫 인상 개선만, 흐름/로직 변화 0

### Stage 2 — 카카오 로그인을 Step 3 직후로 이동 (#26, 2026-04-07 저녁 배포)

- `/onboarding` Step 0~3을 비로그인(anon) 접근 가능
- 사용자가 닉네임 → 성별 → 생년월일 → 생시 입력 후 "사주 결과 보기" CTA에서 카카오 OAuth 트리거
- sessionStorage(`momo_pre_onboarding`, 24h TTL)로 OAuth 왕복 중 form 데이터 보존
- 카카오 로그인 후 사진(Step 4)부터 시작 (sessionStorage에서 form hydration)
- 회원 재방문은 `getOnboardingStep()` 결과대로 `/result` 직행 또는 비어있는 step
- 랜딩 CTA: 바텀시트 → `/onboarding` 직행 (`<Link>` 버튼)
- 개인정보처리방침: 카카오 동의항목 심사 통과형으로 정비

**앱 영향 0** — DB/RLS/Edge Function/Storage/공유 페이지/분석 호출 타이밍 모두 변경 없음.

### 광고 destination URL 변경 옵션 A — 예정 (노아님 결정 대기)

리서치 결과 광고 URL을 직접 변경하면 메타 머신러닝 학습 단계가 리셋되어 4~7일 광고 효율 변동성 + ROAS 20~40% 일시 하락 가능.

→ **권장 옵션 A**: Next.js middleware로 server-side redirect.
- 광고 URL은 `/?utm_source=meta&...` 그대로 유지
- middleware에서 `/` + `utm_source` 조건 → `/onboarding`으로 307 redirect
- 광고 학습 100% 보존 + 1순위 누수도 해결
- 일반 방문자(직접 URL, SEO)는 `/` 그대로

→ 이 옵션은 노아님 결정 후 별도 PR로 진행 예정.

## 데이터 관련 주의사항

### sessionStorage 키
- **`momo_pre_onboarding`**: Stage 2 신규. 24h TTL, 클라이언트 전용.
- **`momo_compat_partner`**: 기존 (공유 페이지 → 궁합 자동 체크). 변경 없음, Stage 2 흐름과 호환.

- INSERT 시점: **Step 4 사진 통과 시점** (Stage 2 전후 동일)
- Stage 2가 INSERT 패턴 자체는 변경하지 않음
- 측정 시 "Stage 2 후 INSERT가 늘었는지"가 핵심 지표

### 카카오 OAuth
- scope/동의항목 변경 없음 (현재 카카오 콘솔 설정 그대로 사용)

- 변경된 개인정보처리방침은 노션에 동기화 필요 (노아님 수동)

### GA4 신규 이벤트

- DebugView에서 발화 검증 가능
