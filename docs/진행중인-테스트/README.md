# 진행중인 테스트 (Experiments)

momo-web에서 진행 중이거나 측정 중인 가설 기반 테스트들을 데이터 중심으로 기록하는 폴더입니다.

## 목적

- 모든 product/growth 변경을 **가설 → 변경 → 측정 → 결론**의 사이클로 추적
- 같은 실수를 반복하지 않기 위한 의사결정 히스토리
- 다음 이터레이션의 baseline 데이터 확보
- 효과 없는 변경은 빠르게 롤백, 효과 있는 변경은 빠르게 확장

---

## 파일 명명 규칙

`YYYY-MM-DD-짧은-설명.md`

예시:
- `2026-04-07-카카오-로그인-누수-해결.md`
- `2026-04-15-광고-카피-AB-테스트.md`
- `2026-04-20-사진-단계-이탈-개선.md`

## 작성/관리 원칙

1. **가설 없는 변경은 기록하지 않는다.** 단순 버그 fix는 git 커밋으로 충분.
2. **숫자 없는 결론은 없다.** "느낌상 좋아진 듯"은 결론이 아님.
3. **롤백 방법을 항상 먼저 적는다.** 시작 단계에서 명시 → 패닉 상황에서 빠르게 복구.
4. **D+3, D+7, D+14 체크 일정을 캘린더에 등록한다.** 안 그러면 잊혀짐.
5. **결론이 나면 종료된 테스트 표로 옮긴다.** 진행 중 표는 깨끗하게 유지.
